# 06-打包与发布

本章描述如何把一个 `.alice` 程序打包成可分发的产物。

## 目标

- 日常开发：产物尽量小，依赖目标机已安装 .NET runtime
- 离线分发：生成严格单文件 `.exe`，可在没有 .NET runtime 的机器上运行

## package 命令

```bash
./alice package <file.alice> [-o <outDir>] [--mode minimal|exe|single|both] [--rid <rid>] [--self-contained]
```

示例：

```bash
# 输出两套：minimal + single
./alice package examples/bubble_sort.alice --mode both -o artifacts/bubble_sort.pkg

# 仅输出 minimal（小，依赖系统 runtime）
./alice package examples/bubble_sort.alice --mode minimal

# 仅输出 single（单文件，默认依赖系统 runtime）
./alice package examples/bubble_sort.alice --mode single

# 输出 single 自包含（单文件 + 运行时，适合 Win7/老系统）
./alice package examples/bubble_sort.alice --mode single --self-contained
```

### mode 说明

- `minimal`：输出 `.dll` + `.runtimeconfig.json` + `Alice.Std.dll`，并生成 `xxx.ps1/xxx.sh` 启动脚本（体积小，适合 Win10/11 已安装运行时的场景）
- `exe`：输出可双击运行的 `xxx.exe`（不打包运行时），同时输出 `xxx.dll/.runtimeconfig.json/Alice.Std.dll`（更接近 VS 发布的 framework-dependent apphost 体验）
- `single`：输出严格单文件可执行文件（默认依赖系统 runtime；配合 `--self-contained` 可变为离线可运行）
- `both`：同时输出 `minimal` 与 `single`

### rid 说明

`--rid` 用于 `single` 模式生成对应平台的可执行文件。
常见值：

- Windows x64：`win-x64`
- Linux x64：`linux-x64`
- macOS x64：`osx-x64`
- macOS ARM：`osx-arm64`

如果你只在 Windows 上使用，可以保持默认 `win-x64`。

## 标准库（std）定位规则

如果你在仓库外运行编译器，仍然可以正常 `import io as io` 这类标准库模块。编译器按以下优先级查找 `std/`：

1. `--module-path <dir>`（你手动指定的模块根）
2. 环境变量 `ALICE_STD` 指向的目录（应该是包含 `io.alicei` 的那个 `std` 目录）
3. 以可执行文件所在目录为起点向上探测 `std/`
4. 当前工作目录下的 `./std`

## 输出目录结构

执行：

```bash
./alice package examples/bubble_sort.alice --mode both -o artifacts/bubble_sort.pkg
```

将得到类似结构：

```
artifacts/bubble_sort.pkg/
  minimal/
    bubble_sort.dll
    bubble_sort.runtimeconfig.json
    bubble_sort.cmd
  single/
    bubble_sort.exe
```

如果使用 `--self-contained`，单文件产物会在：

```
single-selfcontained/
  bubble_sort.exe
```

## TUI 模式

可以用 TUI 菜单快速选择命令：

```bash
./alicec -tui
```

说明：TUI 是终端交互菜单，适合快速试用。

### TUI 入口自动检测

在 TUI 中，当你被提示输入 `.alice/.ailice` 路径时：

- 如果你直接回车留空，编译器会扫描当前工作目录（不递归）下所有 `.alice` 与 `.ailice` 文件
- 如果恰好只有一个文件包含 `fun main`（或 `async fun main`），则自动把它当作入口执行
- 如果找到多个入口文件，会列出候选并要求你显式输入路径
- 如果一个都找不到，也会给出已扫描文件列表

### TUI 工作目录

TUI 顶部会显示“工作目录”。自动检测入口时使用该目录进行扫描。

- 你可以选择 `7) 切换工作目录 (cd)` 来修改它
- 当你手动输入一个文件路径执行 run/build/package 时，TUI 会自动把工作目录切换到该文件所在目录
